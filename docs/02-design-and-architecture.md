# 02 – Проектиране и архитектура на решението

## 1. Тип и структура на приложението

Проектът се състои от **две отделни приложения**, които работят заедно:

### 1.1 Duckmension-VS (Играта)

**Тип:** Десктоп / конзолно приложение (игра)

Играта е написана изцяло на **Lua** с помощта на фреймуърка **LÖVE 12.0**.
**Структура на проекта:**

```
Duckmension-VS/
├── main.lua              # Входна точка на играта
├── conf.lua              # Конфигурация на LÖVE
├── const.lua             # Глобални константи
├── gui.lua               # Интерфейс (HUD)
│
├── Duck.lua              # Играч / патица
├── Enemy.lua             # Противници
├── Animation.lua         # Анимации
├── Music.lua             # Музика и звуци
├── Network.lua           # Мрежова комуникация
│
├── GameState.lua         # Базово game state
├── TitleState.lua        # Главно меню
├── PlayState.lua         # Игрово ниво
├── SettingsState.lua     # Настройки
├── WinState.lua          # Екран за победа
│
├── Level.lua             # Зареждане на нива
├── World.lua             # Физически свят
│
├── Floor.lua             # Стандартен под
├── FallingFloor.lua      # Падащ под
├── ToggleFloor.lua       # Превключващ под
├── ReverseToggleFloor.lua
├── Button.lua            # Бутон
├── Button_Floor.lua      # Под-бутон
├── Door.lua              # Врата
├── Portal.lua            # Портал
├── Spikes.lua            # Шипове
├── Trampoline.lua        # Батут
├── Pellet.lua            # Проектил
├── PelletLauncher.lua    # Изстрелвач на проектили
│
├── baton.lua             # Библиотека за контроли
├── assets.lua            # Ресурси (Windows)
├── assets32.lua          # Ресурси (3DS)
├── assets_duck.lua       # Ресурси за патицата
├── assets_shared.lua     # Споделени ресурси
├── assets_tiles.lua      # Тайлове
│
├── assets/               # Изображения, звуци
├── levels/               # Файлове на нивата (50 нива)
├── extra/                # Лого и допълнителни файлове
├── network/              # Мрежови помощни файлове
├── server/               # Сървърен код
└── nest-master/          # Библиотека nëst (конзолно тестване)
```

---

### 1.2 Duckmension-VS-Website (Уеб сайтът)

**Тип:** Уеб приложение (ASP.NET)

Уеб сайтът е написан на **C#** с **ASP.NET** и служи за управление на потребители, показване на класации и реклама на играта. Комуникира с играта чрез **Luasocket**.

**Структура на проекта:**

```
Duckmension-VS-Website/
├── Duckmension-Website.slnx        # Solution файл
├── dotnet-tools.json               # .NET инструменти
├── .gitignore / .gitattributes
├── README.md
└── Duckmension-Website/            # Основен проект
    ├── Controllers/                # ASP.NET контролери
    ├── Models/                     # Модели на данните
    ├── Views/                      # HTML изгледи (Razor)
    └── wwwroot/                    # Статични файлове (CSS, JS)
```

## 2. Модел на данните

### 2.1 Игра (Duckmension-VS)

Данните в играта се управляват чрез Lua обекти (таблици). Основните модели са:

| Клас / Обект     | Описание                                                         |
|------------------|------------------------------------------------------------------|
| `Duck`           | Играчът – позиция, скорост, анимация, козметика, брой победи    |
| `Enemy`          | Противник – позиция, тип, поведение                             |
| `Level`          | Ниво – тайлове, обекти, начална позиция                          |
| `World`          | Физически свят – гравитация, колизии                             |
| `GameState`      | Базово състояние на играта                                       |
| `PlayState`      | Активна игра – текущо ниво, играчи, таймер, резултат            |
| `TitleState`     | Главно меню – избор на режим                                     |
| `SettingsState`  | Настройки – контроли, звук                                       |
| `WinState`       | Финален екран – победител, спечелени cookies                     |
| `Network`        | Мрежова сесия – играчи в лобито, IP, порт, encryption key       |
| `Animation`      | Кадри, скорост, текущ кадър                                      |
| `Music`          | Музикален трак, сила на звука                                    |

**Елементи на нивото:**

| Обект              | Описание                                      |
|--------------------|-----------------------------------------------|
| `Floor`            | Стандартна платформа                          |
| `FallingFloor`     | Платформа, която пада след стъпване           |
| `ToggleFloor`      | Превключваща платформа (вкл./изкл.)           |
| `Button`           | Бутон, активиращ механизми                    |
| `Door`             | Врата, отваряща се с бутон                   |
| `Portal`           | Телепорт между две точки                      |
| `Spikes`           | Шипове – убиват патицата при контакт          |
| `Trampoline`       | Батут – изстрелва патицата нагоре             |
| `Pellet`           | Проектил от изстрелвача                       |
| `PelletLauncher`   | Автоматичен изстрелвач на проектили           |

---

### 2.2 Уеб сайт (Duckmension-VS-Website)

Уеб сайтът работи с потребителски данни. Основните модели включват:

| Модел       | Основни полета                                          |
|-------------|----------------------------------------------------------|
| `User`      | Id, Username, PasswordHash, Email, Cookies, Cosmetics   |
| `Session`   | Id, UserId, Token, CreatedAt, ExpiresAt                 |
| `Leaderboard` | Id, UserId, Wins, Losses, TimeBest                    |
| `Cosmetic`  | Id, Name, Price, ImageUrl                               |

## 4. Потребителски поток

### 4.1 Поток в играта

```
Стартиране на играта
        │
        ▼
┌──────────────┐
│ Главно меню  │ ── Singleplayer ──▶ Избор на ниво ──▶ PlayState
│ (TitleState) │                                            │
│              │ ── Multiplayer ──▶ Свързване към сървър    │
│              │                          │                 │
│              │ ── Settings ──▶ Настройки на контролите   │
└──────────────┘                                            │
                                                            ▼
                                                   Играч минава нивото
                                                            │
                                                            ▼
                                                    WinState (победа)
                                                    Печелят се cookies
                                                            │
                                                            ▼
                                               Следващо ниво / Главно меню
```

**Основни действия на играча:**
- Движение наляво/надясно, скачане
- Взаимодействие с бутони и врати
- Избягване на шипове, проектили и врагове
- Достигане на портала за край на нивото
- В мултиплейър – надпревара с другите играчи

---

### 4.2 Поток в уеб сайта

```
Начална страница (Реклама на играта)
        │
        ├──▶ Регистрация / Вход
        │           │
        │           ▼
        │    Потребителски профил
        │    (статистики, спечелени cookies, козметика)
        │
        └──▶ Класация (Leaderboard)
                    │
                    ▼
             Топ играчи по победи / най-добро време
```

**Основни екрани:**
- **Начална страница** – описание и реклама на играта
- **Вход / Регистрация** – автентикация на потребителя
- **Профил** – брой победи, cookies, притежавана козметика
- **Класация** – глобална класация на играчите

## 5. Използвани технологии

### 5.1 Duckmension-VS (Играта)

| Технология        | Роля                                      | Защо е избрана                                               |
|-------------------|-------------------------------------------|--------------------------------------------------------------|
| **Lua**           | Основен език на играта                    | Лек, бърз скриптов език, идеален за игри с LÖVE              |
| **LÖVE 12.0**     | Game framework                            | Прост, мощен фреймуърк с вградена физика, аудио и графика   |
| **Luasocket**     | Мрежова комуникация (мултиплейър)         | Стандартна Lua библиотека за TCP/UDP сокети                  |
| **Baton**         | Управление на контролите                  | Абстрахира клавиатура, мишка и контролер в едно              |
| **Lovepotion**    | Порт за Nintendo 3DS                      | Позволява компилиране за конзоли без пренаписване на кода    |
| **nëst**          | Тестване на конзолна съвместимост         | Бързо тестване без реална конзола                            |

---

### 5.2 Duckmension-VS-Website (Уеб сайтът)

| Технология        | Роля                                      | Защо е избрана                                               |
|-------------------|-------------------------------------------|--------------------------------------------------------------|
| **C# / ASP.NET**  | Backend логика и контролери               | Мощен, типизиран език с отличен уеб фреймуърк                |
| **Razor (HTML)**  | Шаблони за изгледи                        | Интегриран в ASP.NET, позволява динамично HTML генериране    |
| **CSS**           | Стилизиране на страниците                 | Стандарт за уеб дизайн                                       |
| **JavaScript**    | Клиентска интерактивност                  | Необходим за динамични елементи в браузъра                   |
| **SQL база данни**| Съхранение на потребители и статистики    | Надеждно и структурирано съхранение на релационни данни      |
| **Luasocket**     | Комуникация между играта и сайта          | Директна връзка с вече използваната мрежова библиотека       |
